Implementation Overview

Feature Flag-Driven Recovery

// NEW: Custom CLI flag I added to toggle auto-recovery behavior
omni ocld test xdc m s \
--from ${source} --to ${target} \
-n ${namespace} \
--enable-failure-handling=true // ← enables auto-recovery!

What This Flag Does

  • ✅ Allows automatic failure detection
  • ✅ Generates comprehensive status reports via consistency checks across planes and replicas
  • ✅ Sends rich context via Slack notifications
  • ✅ Enables smart retry logic
  • ❌ Without it: Falls back to manual investigation flow

Critical Failure Points Covered

Step 1

connectToTargetCluster

Network, auth, availability

Step 2

prepareReplica

Conflicts, resources, config

Step 3

callFailoverNamespace

Timeouts, state, conflicts
